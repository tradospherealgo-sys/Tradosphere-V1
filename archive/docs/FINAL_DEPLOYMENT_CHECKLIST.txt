STEP 2: Deploy to Railway (Backend)
--------
# 1. Login to Railway
railway login

# 2. Set environment variables in Railway Dashboard:
#    Go to: Project → Settings → Variables
#    Add these variables:
#
#    FLASK_ENV=production
#    FLASK_SECRET_KEY=<generate_with_command_below>
#    ANGEL_ONE_API_KEY=<your_key>
#    ANGEL_ONE_CLIENT_CODE=<your_code>
#    ANGEL_ONE_PIN=<your_pin>
#    ANGEL_ONE_TOTP_SECRET=<your_secret>
#    DATABASE_URL=postgresql://user:password@host:5432/tradosphere
#    LOG_LEVEL=INFO
#    WORKERS=4

# Generate FLASK_SECRET_KEY:
python3 -c "import secrets; print('FLASK_SECRET_KEY=' + secrets.token_hex(32))"

# 3. Deploy via git push (auto-triggers Railway deployment)
git push origin main

# 4. Monitor deployment
railway logs --follow

# 5. Verify deployment
curl https://tradosphere-backend.railway.app/api/health


STEP 3: Deploy to Vercel (Frontend)
--------
# 1. Login to Vercel
vercel login

# 2. Set environment variable in Vercel Dashboard:
#    Go to: Project → Settings → Environment Variables
#    Add:
#    Name: NEXT_PUBLIC_API_URL
#    Value: https://tradosphere-backend.railway.app
#    Environments: Production

# 3. Deploy via git push (auto-triggers Vercel deployment)
git push origin main

# 4. Monitor deployment
vercel logs

# 5. Verify deployment
curl https://tradosphere-frontend.vercel.app/

SUPPORT & TROUBLESHOOTING
================================================================================

Common Issues:

Issue: "Cannot reach backend"
  Solution: Check health endpoint
    curl https://tradosphere-backend.railway.app/api/health
  Solution: Check Railway logs
    railway logs
  Solution: Verify environment variables set
    railway env

Issue: "Database connection error"
  Solution: Verify DATABASE_URL environment variable
    railway env | grep DATABASE_URL
  Solution: Test PostgreSQL connection locally
    psql $DATABASE_URL -c "SELECT 1"
  Solution: Check db_init.py ran successfully
    railway logs | grep "DATABASE INITIALIZATION"

Issue: "Broker authentication failed"
  Solution: Verify Angel One credentials
    railway env | grep ANGEL_ONE
  Solution: Check Angel One API status
  Solution: Verify TOTP secret is correct

Issue: "Dashboard shows blank page"
  Solution: Check browser console (F12)
  Solution: Check Vercel logs
    vercel logs
  Solution: Verify NEXT_PUBLIC_API_URL is set
    vercel env

Critical Issue - Need Immediate Help:
  1. Check health endpoints
  2. Review logs (railway logs, vercel logs)
  3. Check environment variables
  4. If still stuck, rollback to previous version
  5. Contact support:
     - Railway: support@railway.app
